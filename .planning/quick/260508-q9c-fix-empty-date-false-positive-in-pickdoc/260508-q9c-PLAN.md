<objective>
Fix empty-date false-positive in `pickDocumentStatus`.

**The bug:** In `internal/ems/parse.go:166-170`, `tbl.Interpret(cellBytes, col)` for a dBASE date column (type `D`, length 8) whose raw bytes are 8 spaces returns a `time.Time` zero value. `fmt.Sprint(val)` then yields `"0001-01-01 00:00:00 +0000 UTC"` — a non-empty string. Stored in `bundle.AD2["DATE_OUT"]`, this defeats the `lookup(b.AD2, "DATE_OUT") != ""` check in `pickDocumentStatus`, so every fresh estimate is misclassified as a closed RO and emits `DocumentStatus="C"` → server schedules a `review` SMS job instead of a `24h`/`3day` estimate follow-up.

**Confirmed false positives (already in jobs.db):** Job 113 (Russell Rosete), Job 115 (Jason Rigor), Jobs 150-154 (May 11 pending).

**The fix:** In `readFields`, type-assert the `Interpret` result. If it is a `time.Time` and `t.IsZero()`, store `""` instead of `fmt.Sprint(val)`.

**Scope:**
- One-line behavioural fix in `parse.go` (plus a `time` import).
- One regression test in `parse_test.go` (parse-level: empty date column → "").
- One regression test in `bms_test.go` (closed-RO override negative case for empty DATE_OUT).
- The uncommitted working-tree edit to `bms.go` (RO_CMPDATE removed from close detection) is **preserved** and committed alongside this fix — that is the correct upstream policy and pairs with the parse.go fix.

**Out of scope:** Cancelling false-positive jobs in jobs.db, rebuilding the installer, the unrelated UI/admin working-tree changes, .gitignore changes, and the other `quick/*-PLAN.md` artifacts already on disk.

Purpose: Stop misclassifying every fresh estimate as a closed RO. Eliminate spurious review SMS jobs at the source.
Output: Fixed `parse.go`, two regression tests, and the preserved `bms.go` edit committed atomically.
</objective>
