## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] go-winres v0.3.3 schema differs from plan's template**
- **Found during:** Task 2 (running go-winres make)
- **Issue:** Plan specified winres.json with keys "1", "0000" for RT_MANIFEST/RT_VERSION and nested "0000" language IDs. go-winres v0.3.3 actually uses "#1" for resource IDs and "0409" (English US LCID) for info language sections. Running with the plan's schema produced "invalid language identifier" error.
- **Fix:** Ran `go-winres init` to get the actual schema, then populated with correct values. Confirmed by successful `go-winres make` and go-winres extract showing all fields correctly embedded.
- **Files modified:** winres/winres.json
- **Verification:** `go-winres make` exits 0; `go-winres extract` from built exe shows ProductName, CompanyName, FileVersion, manifest all correct
- **Committed in:** 1d64d6f (Task 2 commit)

**2. [Rule 1 - Bug] .syso must be in cmd/earlscheib/, not project root**
- **Found during:** Task 2 (make build-windows test)
- **Issue:** go-winres outputs rsrc_windows_amd64.syso to cwd (project root). Go build `./cmd/earlscheib` only auto-links .syso files in the package directory (cmd/earlscheib/). The resource was not being embedded — verified by `strings` and `go-winres extract` showing no version info in exe.
- **Fix:** Added `mv -f rsrc_windows_amd64.syso cmd/earlscheib/rsrc_windows_amd64.syso` and `rm -f rsrc_windows_386.syso` to generate-resources target. Updated clean and .gitignore accordingly.
- **Files modified:** Makefile, .gitignore
- **Verification:** `go-winres extract dist/earlscheib.exe` shows ProductName "Earl Scheib EMS Watcher", CompanyName "Earl Scheib Auto Body Concord", FileVersion 0.1.0.0, manifest with minimum-os win10, asInvoker UAC level
- **Committed in:** 1d64d6f (Task 2 commit)

**3. [Rule 1 - Bug] Icon path in winres.json is relative to winres/ directory**
- **Found during:** Task 2 (first go-winres make attempt)
- **Issue:** Plan specified `"icon": "winres/icon.png"` in RT_GROUP_ICON. go-winres resolves icon paths relative to the JSON file's directory (winres/), not the project root, resulting in "open winres/winres/icon.png: no such file" error.
- **Fix:** Changed icon path to `"icon": "icon.png"` (relative to winres/).
- **Files modified:** winres/winres.json
- **Verification:** go-winres make exits 0, icon embedded in exe (APP_0000.ico present in go-winres extract output)
- **Committed in:** 1d64d6f (Task 2 commit)
