<objective>
Configure go-winres to embed Windows version info (ProductName, FileVersion, CompanyName), an application manifest (UAC asInvoker level, Windows 10 compatibility GUID), and a placeholder icon into the `earlscheib.exe` binary. The `.syso` file is generated as a pre-build step and linked automatically by `go build`.

Purpose: Without this, the binary's Properties > Details tab is blank (fails Phase 1 success criterion 5) and Windows heuristics rate the binary as higher risk. The manifest prevents unnecessary UAC prompts when the user runs `earlscheib.exe` directly. All of this must be in place before signing (Plan 04) because the signed binary must already contain correct metadata.

Output: `winres/winres.json`, `winres/icon.png` (placeholder), `rsrc_windows_amd64.syso` (generated; gitignored), updated `Makefile`.

**File 2: `winres/icon.png`**

Generate a minimal valid 256x256 PNG placeholder using Go's `image` stdlib (do not require any external tool). Write a small Go script `winres/gen_icon.go` (with build tag `//go:build ignore`) that creates a solid green 256x256 PNG and saves it to `winres/icon.png`. Then run it once with `go run winres/gen_icon.go`.

The gen_icon.go content:

```go
//go:build ignore

package main

import (
    "image"
    "image/color"
    "image/png"
    "os"
)

func main() {
    img := image.NewRGBA(image.Rect(0, 0, 256, 256))
    green := color.RGBA{R: 34, G: 139, B: 34, A: 255} // forest green
    for y := range 256 {
        for x := range 256 {
            img.Set(x, y, green)
        }
    }
    f, err := os.Create("winres/icon.png")
    if err != nil {
        panic(err)
    }
    defer f.Close()
    if err := png.Encode(f, img); err != nil {
        panic(err)
    }
}
```

Run: `go run winres/gen_icon.go`

This produces `winres/icon.png` (a solid green square — placeholder only; the real icon is added in Phase 3/4).
