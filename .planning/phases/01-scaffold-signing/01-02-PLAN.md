  <action>
Create `.github/workflows/build.yml`. The workflow must:

1. Trigger on every `push` to any branch and on every `pull_request`.
2. Run on `ubuntu-latest` (no Windows runner).
3. Use `actions/setup-go@v5` with Go version matching `go.mod` (1.22 or higher).
4. Cache Go module downloads with `actions/cache@v4` using the `go.sum` hash as the cache key.
5. Run `go vet ./...` before building.
6. Run `make build-windows` with `GSD_HMAC_SECRET` sourced from `secrets.GSD_HMAC_SECRET`. If the secret is absent (e.g., forks without the secret set), the build still succeeds using the dev-default in source.
7. Upload `dist/earlscheib.exe` as a workflow artifact named `earlscheib-windows-amd64` with a 7-day retention.
8. Do NOT include any signing step yet — Plan 04 adds that.

Use this exact structure:

```yaml
name: Build

on:
  push:
    branches: ["**"]
  pull_request:
    branches: ["**"]

jobs:
  build-windows:
    name: Cross-compile windows/amd64
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
          cache: true

      - name: Vet
        run: go vet ./...

      - name: Build windows/amd64
        env:
          GSD_HMAC_SECRET: ${{ secrets.GSD_HMAC_SECRET }}
        run: make build-windows

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: earlscheib-windows-amd64
          path: dist/earlscheib.exe
          retention-days: 7
          if-no-files-found: error
```

Create the `.github/workflows/` directory if it does not exist.

NOTE: CGO_ENABLED=0 is set in the Makefile target — no additional `apt-get install gcc-mingw-w64-x86-64` step is needed in this workflow. Plan 03 (signed Phase 3 CGO build) will add that when systray is introduced.
