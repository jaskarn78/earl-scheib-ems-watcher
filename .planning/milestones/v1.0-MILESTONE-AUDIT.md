# v1.0 Milestone Audit — Earl Scheib EMS Watcher (Windows Client)

## Status: tech_debt (post-fix)

All 41 active requirements satisfied. One BLOCKER was discovered and **fixed inline during audit** (admin proxy double-path URL). No remaining code-level blockers. Accumulated tech debt is bounded to HUMAN-UAT items (Windows VM for Phase 3, real-browser testing for Phase 5) and external dependencies (OV code-signing cert).

## Summary of Milestone

Five phases shipped:
1. **Phase 1 — Scaffold + Signing** — Go module, CI cross-compile from Linux, go-winres resource embed, osslsigncode signing pipeline (OV cert procurement is the only external gate).
2. **Phase 2 — Core Scanner** — Pure-Go, CGO-free port of the Python reference watcher. Dedup, settle check, HMAC-SHA256 request signing, retry with exponential backoff, heartbeat, slog+lumberjack logging.
3. **Phase 3 — Installer + Native Config** — Inno Setup single-exe installer with native folder-picker wizard, icacls ACL hardening, Scheduled Task registration as SYSTEM (with user-account fallback for mapped drives).
4. **Phase 4 — Telemetry + Remote Config** — telemetry.Wrap catches panics and posts minimal records; remote-config fetch at start of every --scan merges whitelisted keys atomically into config.ini; matching server endpoints on app.py.
5. **Phase 5 — Queue Admin UI** — `earlscheib.exe --admin` opens a local browser UI (Concord Garage aesthetic: Fraunces + JetBrains Mono + oxblood palette) that lists pending SMS jobs grouped by customer; Marco can cancel with 5s optimistic undo. HMAC-signed proxy between browser and remote `/queue` endpoint.

### Fix Applied Inline

**BLOCKER (discovered 2026-04-21 during audit):** `internal/admin/server.go` `remoteQueueURL()` appended `/earlscheibconcord/queue` to WebhookURL which already ends in `/earlscheibconcord`, producing a double-path 404 in production. Tests passed only because `httptest.NewServer` accepts any path.

**Fix:** Changed to append `/queue` only (commit `bbf1f09`). Tests updated to pass `remote.URL + "/earlscheibconcord"` as the WebhookURL so production URL shape is exercised. Assertions on upstream path (`/earlscheibconcord/queue`) remain valid. Build + test suite green.
