### Gaps Summary

One gap blocks full CI pipeline certification. All local build artifacts are correct and functional:

**Gap — CI missing go-winres installation (SCAF-02):** The CI workflow calls `make build-windows` which depends on `generate-resources` which calls `go-winres make`. The `go-winres` binary is not pre-installed on `ubuntu-latest` GitHub Actions runners and is not installed by `actions/setup-go`. The Makefile provides a `make install-tools` target for this purpose but it is not called in the workflow. Without this, every CI push will fail at the `generate-resources` step with exit 127.

Fix: Add one step to `.github/workflows/build.yml` between "Set up Go" and "Vet":
```yaml
      - name: Install go-winres
        run: go install github.com/tc-hib/go-winres@v0.3.3
```
